O que é REST?

REST é uma técnica de engenharia de software para sistemas distribuídos.

Surgiu em 2000, em uma tese de doutorado sobre a web, escrita pelo norte-americano Roy Fielding *

* um dos principais autores da especificação HTTP

Principais características

  • Utiliza padrões já existentes
  • Stateless
  • Orientado a recurso
  • Interface uniforme para acesso aos recursos (URI)
  • Representações independentes

REST - Stateless

Toda requisição HTTP é completamente isolada das outras

Cada requisição precisa incluir todas as informações para uma resposta do servidor

REST - Recursos

Tudo que é importante ao ponto de ser exposto na web

  • Exemplos:
    • Tweets no Twitter
    • Post no Facebook
    • Fotos no Instagram

Todo recurso possui no mínimo uma URI

REST - Interface uniforme

Utiliza os métodos do protocolo HTTP para manipulação de recursos (Verbos)

  • GET
  • POST
  • PUT
  • DELETE

REST - GET

Utilizado para obter uma representação do recurso

  • Exemplo:
    • www.bookstore.com/books/history

REST - POST

Utilizado para criar um novo recurso

  • Exemplo:
    • www.bookstore.com/books/new

REST - PUT

Utilizado para modificar um recurso existente

  • Exemplo:
    • www.bookstore.com/books/edit{id}

REST - DELETE

Utilizado para deletar um recurso

  • Exemplo:
    • www.bookstore.com/books/delete{id}
